Make a program that implements a simple, thread-safe caching system in Kotlin. This system should have the following features:

  1. Tracking Hit Count: The system should record the total number of times someone accessed the cache, including both reads and writes.

  2. Caching Operations: Perform two main tasks: adding an item to the cache and getting an item from the cache. While adding an item, update the value if the key already exists.

  3. Supporting Multithreading: This caching system must be secure in a multithreaded environment. It should work correctly even when receiving simultaneous access from different threads, while preventing data races and other multithreading issues.

  4. Testing the System: Show an example of how to use the caching system with multiple threads. Each thread should be adding and retrieving items from the cache. At the end of the program, display the total number of cache accesses.

You should use atomic variables val hitCount and thread-safe collections in Kotlin val cache in your solution.



Memory limit: 256 MB
Time limit: 8 seconds
